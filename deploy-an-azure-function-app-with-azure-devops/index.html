Deploy an Azure Function App with Azure Devops

Reuben Cleetus

Import Source code to DevOpsBuilding a basic Azure function is a pretty simple process, but the real value in the Azure tooling is that deploying even a complex multi-function app is straightforward with Azure DevOps. I'll show you how to do that here, and you'll see how easy it is to wire up a CI/CD pipeline to your Azure Function App code.

Step 1 - (Optional) Import source code into your new Project

This step is Optional. As you'll see in Step 2, you can work just as easily with code in another repository (such as Github); however, DevOps does offer you a very easy collaboration and project management space, so it's certainly worth a look.

Create a new Project in Azure DevOps, and then Import your source code into DevOps by clicking on Files, and then using one of the various options to get your code into DevOps. I often work in Github, so if I'm working with others on Azure DevOps, I'll just add another remote to my Git repo that I call devops, and push to it. I'll assume you're using .NET Core.

Step 2 - Create a Build Pipeline

In this step, we'll create a build pipeline that takes the source you just imported into DevOps, and produces a compiled app.

  • Hover over Pipelines and click on Builds (or click on Pipelines, and in the expanded menu, click on Builds)
  • Click New pipeline
  • I like working in the Classic Editor, so click on use classic editor
  • Point DevOps to your code
  • On the Select Template screen, search for "Core", and select ASP.NET Core, and click Apply. This sets up a build step that builds your code assuming an ASP.NET Core App.
  • Note that when you click on the Publish Artifact Step, the compiled output is placed into an output artifact called drop.

And that's it for Building!

 

Step 4 - Deploy the compiled artifacts

This step will take your compiled artifacts and deploy it to the App Service Plan in your Function App. Recall that a Function App, even one on the Consumption Plan where you pay only when your function runs, actually creates an instance of an App Service Plan (ASP). This ASP is what you'll deploy your App to.Select Azure App Service Deployment

Use the following settings for the stage:
  • Stage Name: Any name you'd like for the pipeline stage. It has no impact on the deployed application
  • Azure Subscription: Select your subscription
  • App Type: Function App on Windows
  • App Service Name: Select the Function App's ASP that you provisioned

That's it. You should be set up and ready to deploy your function app.
